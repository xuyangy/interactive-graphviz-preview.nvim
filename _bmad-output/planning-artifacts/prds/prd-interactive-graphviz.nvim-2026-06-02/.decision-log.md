## Decisions
- 2026-06-02 — Working mode: **Fast path** (draft + [ASSUMPTION] tags, iterate).
- 2026-06-02 — **Platform binaries:** Linux + macOS, x64 + arm64 (4 prebuilt binaries). **Windows deferred** to a later release (covered in interim by source-build fallback).
- 2026-06-02 — **Render engine:** Bundle **Graphviz-WASM** (`d3-graphviz` + `@hpcc-js/wasm`), zero system dependencies — no system `dot`/graphviz required. Same renderer as the VSCode reference. System-`dot` hybrid path explicitly deferred.
- 2026-06-02 — **Minimum Neovim version: 0.10+** (for stable `vim.system()` async process spawning).
- 2026-06-02 — **Build-from-source fallback: Yes.** When no prebuilt binary matches the platform, fall back to building from source (requires Bun). This is how Windows/BSD/oddball-arch users run it until prebuilt coverage expands.
- 2026-06-02 — Transport/mechanism detail (initially msgpack-RPC, WebSocket, server lifecycle) → captured in `addendum.md` (architecture-bound), kept out of the capability-level PRD. Superseded by architecture update below: v1 uses newline-delimited JSON over `vim.system()` stdio.
