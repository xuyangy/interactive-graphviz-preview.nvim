## What Went Well

1. **Fail-closed integrity end-to-end.** Download → SHA-256 verify against committed `checksums.txt` → atomic rename → `chmod +x`. A tampered/truncated download is never promoted or run. The in-source manifest is the trust root, never a mutable remote file beside the asset.
2. **No-orphan invariant survived the new fallback path.** The source-build fallback compiles a single-file executable and returns *that* path — no `bun run` wrapper — so `server.lua`'s `vim.system(stdin=true)` EOF/heartbeat supervision is unchanged. The hardest cross-cutting guarantee of the project was explicitly preserved, with a live-route test asserting `/health` never writes to the stdout protocol channel.
3. **Reviews caught real bugs every story.** 3.1: release helper resolved build inputs from caller `cwd` → fixed to repo root (would have produced mismatched checksums). 3.2: verified cache-hit skipped `chmod`/quarantine prep → fixed. These were genuine defects, not nits.
4. **Strong scope discipline.** Every story carried an explicit "do NOT implement X — belongs to Story Y" boundary and a "Previous Story Intelligence" section. Story 3.2 consumed exactly the artifact names 3.1 produced; 3.3 converted 3.2's intentional hard-fail into fallback without weakening it. Clean handoffs by construction.
5. **A recurring pain was turned into a reusable asset.** "busted not installed locally" appeared in all three stories. The response matured from a bare note (3.1) → ad-hoc compatibility runner (3.2) → a committed `tests/support/busted_compat.lua` shim (3.3) that runs the Lua specs under plain `lua`. The team productized its own friction.
6. **Honest validation reporting.** Where something could not be run (busted locally; the compiled fallback binary end-to-end under the sandbox), it was stated plainly rather than papered over.

## What Was Harder

1. **Local test parity is approximate.** `busted` is not installed locally, so the canonical Lua gate lives only in CI. The local shim is good, but it is a re-implementation of the runner, not the runner.
2. **Architectural tension in 3.1 (checksums).** "Generate checksums on tagged release" vs "checksums committed in-source" genuinely conflict — a tag workflow can't rewrite the tag's tree. Resolved deliberately (pre-tag manifest-prep mode + verify-on-tag-publish), but it took explicit design thought to avoid silently weakening integrity.
3. **Environmental blockers ate time.** 3.1: Bun cross-compile target-runtime downloads failed under restricted network until access was approved; a stray smoke binary tripped fail-closed validation. 3.3: the `--compile` output binary could not be spawned end-to-end (sandbox-denied) — only built and indirectly verified.
4. **A mixed-model handoff left a latent crash.** 3.3's partial WIP still called the renamed `detect_platform()` from `resolve_server_cmd` — it would have crashed on every invocation. Caught and fixed during 3.3 dev, but it slipped past the point of handoff between implementers.

## Cross-Story Patterns

- **`cwd`-relative path assumptions are a repeat offender** (3.1 review fix; flagged forward in 3.2 and 3.3 dev notes). Lesson was applied forward each time — derive paths from runtime/plugin root.
- **"Preparation must run on the cache-hit path too"** (3.2 review fix) — verification passing is not the same as the binary being spawn-ready. 3.3 explicitly preserved this.
- **Continuity discipline worked.** Because each story documented what it was deferring and to whom, no scope leaked and no story re-litigated a prior decision.

## Key Takeaways

1. Documenting *deferrals* ("not here — Story Y owns this") was as valuable as documenting work; it is why three stories composed cleanly with zero scope leakage.
2. Adversarial review earned its keep — it caught a checksum-mismatch-class bug and a spawn-readiness bug that tests had passed over.
3. Turn recurring friction into committed tooling (the busted-compat shim) rather than re-solving it per story.
4. Across a mixed-model handoff, a smoke run at the handoff point would have caught the renamed-function crash earlier — bake that into future handoffs.
